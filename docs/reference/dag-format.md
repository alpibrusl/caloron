# DAG Format

The sprint DAG is stored as `dag.json`. It is generated by the PO Agent at kickoff and is immutable for the duration of the sprint.

## Schema

```json
{
  "sprint": {
    "id": "sprint-2026-04-w2",
    "goal": "Implement user authentication",
    "start": "2026-04-08T09:00:00Z",
    "max_duration_hours": 72
  },
  "agents": [
    {
      "id": "backend-1",
      "role": "backend-developer",
      "definition_path": "agents/backend-developer.yaml"
    }
  ],
  "tasks": [
    {
      "id": "task-1",
      "title": "Implement JWT token generation",
      "assigned_to": "backend-1",
      "issue_template": "tasks/jwt-implementation.md",
      "depends_on": [],
      "reviewed_by": "reviewer-1"
    }
  ],
  "review_policy": {
    "required_approvals": 1,
    "auto_merge": true,
    "max_review_cycles": 3
  },
  "escalation": {
    "stall_threshold_minutes": 20,
    "supervisor_id": "supervisor",
    "human_contact": "github_issue"
  }
}
```

## Fields

### `sprint`

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Unique sprint identifier |
| `goal` | string | Human-readable sprint goal |
| `start` | ISO 8601 | Sprint start timestamp |
| `max_duration_hours` | u32 | Maximum sprint duration |

### `agents[]`

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Unique agent instance ID (e.g., `backend-1`) |
| `role` | string | Agent role type (e.g., `backend-developer`) |
| `definition_path` | path | Path to agent YAML in meta repo |

Multiple agents can share the same role (e.g., two `backend-developer` agents).

### `tasks[]`

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Unique task ID |
| `title` | string | Task title (becomes the GitHub issue title) |
| `assigned_to` | string | Agent ID that handles this task |
| `issue_template` | path | Template for the GitHub issue body |
| `depends_on` | string[] | Task IDs that must complete before this task can start |
| `reviewed_by` | string? | Agent ID that reviews PRs for this task |

### `review_policy`

| Field | Type | Description |
|-------|------|-------------|
| `required_approvals` | u32 | Number of approvals needed to merge |
| `auto_merge` | bool | Merge automatically when approved |
| `max_review_cycles` | u32 | Cycles before supervisor intervenes |

### `escalation`

| Field | Type | Description |
|-------|------|-------------|
| `stall_threshold_minutes` | u32 | Default stall threshold |
| `supervisor_id` | string | Agent ID of the supervisor |
| `human_contact` | string | Escalation method (`github_issue` or `slack`) |

## Validation Rules

The DAG engine validates:

- **No cycles**: Tasks cannot have circular dependencies
- **Valid references**: All `assigned_to`, `reviewed_by`, and `depends_on` values must reference existing agents/tasks
- **No duplicates**: Task IDs and agent IDs must be unique
