# Feedback Format

When an agent completes a task, it posts a structured feedback comment on the issue. This is mandatory — the harness enforces it.

## Format

```yaml
---
caloron_feedback:
  task_id: "issue-42"
  agent_role: "backend-developer"
  task_clarity: 4
  blockers:
    - "Error response format was not specified in the issue"
    - "Dependency on issue #38 was not in the DAG"
  tools_used:
    - "github_mcp"
    - "noether:http_post_json"
    - "noether:json_validate"
  tokens_consumed: 14200
  time_to_complete_min: 47
  self_assessment: "completed"
  notes: "Had to make assumptions about the pagination strategy"
---
```

## Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `task_id` | string | Yes | The task/issue identifier |
| `agent_role` | string | Yes | The agent's role name |
| `task_clarity` | u8 (1-10) | Yes | How clear was the task? |
| `blockers` | string[] | Yes | What blocked or slowed the agent |
| `tools_used` | string[] | Yes | Tools and Noether stages used |
| `tokens_consumed` | u64 | Yes | Total tokens used |
| `time_to_complete_min` | u32 | Yes | Minutes to complete |
| `self_assessment` | enum | Yes | See below |
| `notes` | string | No | Free-form observations |

## Self Assessment Values

| Value | Meaning |
|-------|---------|
| `completed` | Task finished successfully |
| `partial` | Some work done but not all |
| `blocked` | Could not proceed |
| `failed` | Attempted but failed |
| `crashed` | Synthetic — agent process died (generated by daemon) |

## Synthetic Feedback

If an agent process is killed (OOM, SIGKILL), the daemon generates a synthetic feedback comment:

```yaml
---
caloron_feedback:
  task_id: "issue-42"
  agent_role: "backend-developer"
  task_clarity: 0
  blockers:
    - "Agent process terminated unexpectedly (SIGKILL)"
  tools_used: []
  tokens_consumed: 0
  time_to_complete_min: 0
  self_assessment: "crashed"
  notes: "Synthetic feedback generated by daemon."
---
```

## Usage in Retro

The Retro Engine processes all feedback comments to detect:

- Tasks with clarity < 5 (need better specifications)
- Blockers mentioning "dependency" or "not in the DAG" (missing DAG edges)
- Blockers mentioning "tool" + "unavailable" (agent misconfiguration)
- `noether:*` entries in `tools_used` (stage reuse tracking)
